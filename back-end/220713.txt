서버 : 클라이언트에게 네트워크를 통해 서비스를 제공하는 컴퓨터

서버 구축 방법 
1) 서버 호스팅
 - 본인의 컴퓨터를 직접 서버로 구추하는 방법 -> 거의 사용하지 않는 방식
 --> 컴퓨터의 작동 환경을 마음껏 유동적으로 설정 가능
 --> 웹 호스팅에 비해 가격이 비싸다.
 
2) 웹 호스팅
 - 일정 요금을 내고 외부 서버를 임대해 사용하는 방법

3) 클라우드 서버
 - 프로그램으로 가상의 컴퓨터 (서버) 를 하나 생성해 이용하는 방식

 장점 . 짧은 시간 안에 서버를 하나 뚝딱 만들 수 있다.
      . 물리적인 컴퓨터의 한게를 넘어 무한대의 저장 공간을 이용할 수 있다.

* 클라우드 모델 
1) 퍼블릭 클라우드 : (현재는 개인적으로 사용)
2) 프라이빗 클라우드 : 보안성 최고
3) 하이브리드 클라우드 : 1과 2일 합친것, 가장 유동적으로 사용 가능

ip 주소와 도메인

ip란?
-> 인터넷상에서 데이터를 주고받기 위한 통신 규약(약속)

ip 주소
-> 데이터를 주고 받기 위해 사용되는 주소

내 ip 확인하기
 - cmd 열고 ipconfig 입력

공인 ip vs 사설 ip

공인 ip 
 - 외부에 공개되어 있는 ip
 - 공개되어 있기 때문에 누구나 검색, 접근 가능

사설 ip
 - 로컬 ip, 가상 ip 라고 불리며, 외부에서 접근할 수 없는 ip

유동 ip vs 고정 ip

유동 ip 
 - 기기에 고정적으로 ip를 부여하지 않고 ip 갱신주기가 되었을 때 기기를 껐다 켰을 때 남아 있는 ip를 골라 기기에 ip를 할당해주는 것
 ex) 와이파이 공유기

고정 ip
 - 사용자가 직접 ip 주소를 입력해 주소를 설정하는 방식

도메인
- 웹 브라우저를 통해 특정 사이트를 진입할 때, ip 주소를 대신해 사용하는 주소
- 한눈에 파악하기 힘든 ip 주소보다 분명하게 나타내기 위해 사용
ex) 카페24, 호스팅kr 등등

DNS 
 - comain name system
 - ip 주소와 특정 도메인 주소가 같다는 것을 지정해두고, 인터넷 사용자들이 도메인 주소를 검색했을 때 ip 주소로 연결되도록 해주는 것
