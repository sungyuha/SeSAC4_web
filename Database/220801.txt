데이터베이스 
 - 데이터를 저장하는 구조/자료의 모음
 - 데이터의 집합소

파일 시스템 
단점 - 1. 데이터 중복, 2. 데이터 불일치

DBMS
- 파일 시스템이 가진 문제를 해결하기 위해 만들어진 것
- 데이터베이스에 접근하고 이를 관리하기 위해 존재한다.

RDBMS (관계형 데이터베이스)
MySQL

데이터베이스 용어 (중요)
열 (Columm ,속성)
행 (Record, Tuple ,튜블: 중복 되면 안됨)
테이블 (Table, Relation)

Key : 데이터베이스에서 튜플을 찾거나 순서대로 정렬할 때 구분하고 정렬의 기준이 되는 속성

- 기본키 (Primary Key)
 메인 키로 한 테이블에서 특정 튜플을 유잃게 구별할 수 있는 속성
 Null 값과 중복값 불가
- 외래키 (Foreign Key)
 어떤 테이블의 기본키를 참조하는 속성
 속성 이름은 달라도 되는데, 그 안의 값은 동일해야 한다

MySQL
 - 가장 널리 사용되고 있는 관계형 데이터베이스 관리 시스템
 - 오픈 소스
 - 다양한 운영체제에서 사용 가능

SQL 문
- 구조적 질의 언어
- 데이터베이스를 제어하고 관리할 수 있는 목적의 프로그래밍 언어

DDL
 - 데이터를 정의하는 언어
 테이블에 관련 된 정의를 한다

 -- create : 생성, 거의 테이블 생성에 사용한다.
 -- alter : 테이블의 특정 컬럼을 삭제하거나 추가, 변경 할 때 사용하는 명령어
 -- drop : 테이블 삭제
 
DML
 - 데이터베이스의 내부 데이터를 관리하기 위한 언어

 -- select : 데이터를 검색(조회)하기 위해 사용
 -- insert : 데이터 추가
 -- update : 갱신
 -- delete : 삭제


 오름차순 ASC;
 내림차순 DESC;
